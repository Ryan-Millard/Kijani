Kijani Battle Bot Platform Manual
Introduction
The Kijani Project is an open-source ESP32-based battle robot platform designed to make robot combat cheaper, simpler, and more accessible.
The platform combines:
motor drivers
battery charging
WiFi control
Bluetooth support
web interfaces
servo control
custom firmware
filesystem hosted web apps
all on a single compact PCB.
The goal is to allow anyone to:
build a battle bot
customise the controls
experiment with robot designs
create their own web interfaces
learn embedded systems and robotics

History
The Kijani project started in 2024 after an interest in the antweight combat competitions run by the South African Robot League.
At the time:
combat robots were expensive
many separate electronics modules were required
robot control systems were overly complicated
The original design goal was:
“Make a combat robot controller using a single ESP32 board.”
The earliest version used:
Bluetooth control
an Android app made using MIT App Inventor
In 2025 the project evolved further while assisting with the Africa Rover Moot project.
The design changed to:
support all phones through web browsers
simplify mass production
reduce PCB size
improve field usability
Version 2 of the PCB:
reduced board length by 5mm
simplified the regulator system
reduced component count
improved manufacturability

Recommended Motors
Drive Motors
Recommended:
N20 gear motors
Micro metal gear motors
TT motors (lightweight robots only)
Typical Voltage
3V–6V
Recommended Current
under 1A continuous per motor
Important
Stall current can be much higher than running current.
Always test motors before combat use.

Recommended Servos
Recommended:
SG90
MG90S
Metal gear micro servos
Servo Voltage Warning
The boost converter can raise voltage higher than some servos can tolerate.
Always verify:
servo voltage rating
regulator voltage settings
battery voltage
before enabling boost mode.
Incorrect voltage can permanently damage servos.

Firmware Overview
The robot software has two parts.

1. ESP32 Firmware
Written in:
C++
Arduino Framework
Responsibilities:
motor control
web server
WiFi
settings storage
battery monitoring
filesystem management
Programming is done through:
USB serial
VSCode + PlatformIO

2. Web Interface Files
Stored in:
LittleFS filesystem
Written in:
HTML
CSS
JavaScript
These files:
create the control interface
display telemetry
manage settings
communicate with the API
The web interface can be fully customised.

Setup Guide
First Boot
After flashing the firmware:
Power on the board
Wait for the WiFi access point
Connect using a phone or computer
Open the robot webpage

Default Access Point
Document:
default SSID
default password
here once finalised.

Changing Settings
Settings can be modified:
from the web interface
through the API
Examples:
robot name
WiFi password
control settings
custom configuration values

Updating Web Files
Web files can be uploaded:
through USB
through the built-in web uploader
This allows rapid interface development without reflashing firmware.

Factory Reset
If the password is forgotten:
Power off the board
Short the PGM pins
Power on the board
Listen for the factory reset tune
The board will:
erase stored settings
restore defaults

Warning Notes
Voltage Warning
The boost regulator can output voltages unsafe for:
some servos
low-voltage electronics
Always verify voltage compatibility before enabling higher voltages.

Motor Driver Warning
Large motors can exceed driver limits.
Avoid:
stalled motors
oversized drive motors
direct short circuits

Cooling Warning
High current operation may generate heat.
Monitor:
motor drivers
regulators
battery temperature
during testing.
